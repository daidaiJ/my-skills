# Skill: Planning - 项目规划与方案设计

## 工作流程

### 第一阶段：了解业务背景
- 主动询问项目的业务背景、目标用户、核心价值
- 了解当前的技术栈、现有系统、约束条件
- 确认项目的规模、时间要求、预算限制
- 了解团队的技术能力和经验水平

### 第二阶段：明确需求
- 梳理核心功能需求
- 确认非功能性需求（性能、安全性、可扩展性等）
- 识别优先级和里程碑
- 确认验收标准

### 第三阶段：方案设计
基于需求分析，提供 2-3 个可行的技术方案，每个方案包括：
- **方案概述**：简要描述技术选型和架构思路
- **架构设计**：核心模块划分、数据流、关键组件
- **优势**：为什么选择这个方案
- **劣势/风险**：可能遇到的问题和挑战
- **适用场景**：什么情况下最适合这个方案

### 第四阶段：方案确认与讨论
- 详细解释每个方案的差异和权衡
- 回答你的技术疑问
- 根据你的反馈调整方案
- 帮助你做出最终选择

### 第五阶段：最终修正与文档输出
- 根据确认的方案进行细节修正
- 生成完整的项目计划文档
- 写入项目目录下的 `.plan/plan.md` 文件
- 提供实施建议和下一步行动项

### 注意事项

1. **灵活性**：根据项目规模调整方案的复杂度
2. **实用性**：推荐的技术栈应该是当前主流且有良好社区支持的
3. **可操作性**：方案应该包含具体的实施步骤和时间估算
4. **风险意识**：明确指出可能的风险和应对策略
5. **持续改进**：文档应该易于更新和维护
